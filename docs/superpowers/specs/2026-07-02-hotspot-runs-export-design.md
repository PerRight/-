# 고오염 구역 표시 · 다중 탐사 조회 · Excel 요약 내보내기 — 설계

날짜: 2026-07-02
상태: 사용자 검토 대기 (v2 — 임계값·데이터 형식·내보내기 형식 확정 반영)

## 배경

- 드론의 실제 센서는 **pH, EC** 두 가지다 (dashboard_server.py SENSORS).
- `data/measurements.csv` 는 시뮬레이션 플레이스홀더로 탁도만 담고 있고,
  `run_id`(1~8회차)가 있지만 히트맵·대시보드는 전 회차를 합쳐 평균만 보여준다.
- 고오염 구역 표시, 회차별 조회, 수치 요약 파일 내보내기 기능이 없다.

## 확정된 요구사항

1. **오염 판정: 절대 임계값** — `pH < 6.5 또는 pH > 8.5` **또는** `EC > 500 µS/cm`
   이면 해당 셀을 오염으로 판정하고 **빨간색**(#d03b3b)으로 표시한다.
   임계값은 센서 레지스트리에 센서별 상수로 관리한다.
2. **막대그래프 강조**: 대시보드 3D 막대 뷰에서 값 차이가 시각적으로 잘 보이도록
   막대 높이에 비선형(멱함수) 스케일을 적용해 높은 값일수록 경사가 급하게 솟게 한다.
   (호버 툴팁의 수치는 실제 측정값 그대로 표시 — 왜곡은 높이에만 적용.)
3. **내보내기 형식: Excel(xlsx)** — `openpyxl` 을 requirements.txt에 추가.
4. **데이터 형식 통일**: `measurements.csv` 컬럼을
   `run_id, timestamp, x_m, y_m, depth_m, ph, ec` 로 변경.
   실측 데이터를 받을 때도 같은 컬럼 형식으로 교체하면 된다.
5. **측정 프로파일**: 측정 수심은 **0.5 / 1.0 / 1.5 m 세 단계**, 각 수심에서 **1분씩
   측정해 지점당 3분**. 데이터 생성기는 수심당 20초 간격 3표본(1분)을 기록하고,
   대시보드 시뮬레이터는 수심당 체류 시간을 상수로 관리한다
   (데모 기본 3초 — 실제 운용 시 60초로 변경).
   시뮬레이션 오염 플룸의 수심 중심은 3.0 m → **1.0 m**로 옮겨 측정 대역(0.5~1.5 m)
   안에서 오염이 관측되게 한다 (보조 오염원은 1.5 m → 0.5 m).

## 기능 1 — 고오염 구역 표시 (pH·EC 임계값, 빨간색)

### 판정 규칙 (공통)

- 셀별 평균값 기준: `평균 pH가 6.5~8.5 밖` 또는 `평균 EC > 500 µS/cm` → 오염 셀.
- 단일 센서 화면에서는 그 센서의 임계값만 적용해 표시하고,
  요약(보고서·카드 문구)에서는 "둘 중 하나라도 초과" 기준으로 오염 셀을 센다.

### 오프라인 3D 히트맵 (`visualize_heatmap.py`)

- 임계값을 넘는 셀을 **빨간색 다이아몬드 마커 별도 트레이스**("오염 구역" 범례)로 그린다.
  정상 셀은 기존 파랑 램프 유지.
- PNG(matplotlib)도 동일하게 빨강 강조 + 범례.

### 대시보드 (`dashboard.html` + `dashboard_server.py`)

- 센서 레지스트리에 `threshold` 필드 추가 (pH: 범위형 6.5~8.5, EC: 상한형 500).
  `/api/heatmap` 응답에 임계값 정보 포함.
- 셀 뷰: 임계값 초과 셀을 빨간색으로 표시.
- 막대 뷰: 초과 지점의 막대를 빨간색으로 표시.
- 히트맵 카드 상단 요약 한 줄: "오염 셀 N개 · 최고 xx @ (X, Y, 수심)".
- 센서 카드의 현재값도 임계값 초과 시 빨간색으로 표시.

## 기능 2 — 대시보드 막대그래프 경사 강조

- 막대 높이: `h = (v - vmin) / (vmax - vmin)` 를 0~1로 정규화한 뒤 `h^2.5` 멱함수를
  적용하고 다시 값 범위로 되돌려, 낮은 값은 낮게 깔리고 높은 값은 가파르게 솟게 한다.
- 지수(2.5)는 코드 상수 `BAR_EXAGGERATION` 으로 관리 — 조절 용이.
- z축 눈금은 왜곡된 높이와 안 맞으므로 숨기고, 값은 색·툴팁으로 읽게 한다.

## 기능 3 — 다중 탐사(회차) 선택 조회

### 오프라인 3D 히트맵

- 회차별 + 전체 누적 집계를 모두 계산해 **HTML 하나에 내장**, Plotly `updatemenus`
  드롭다운 2개로 전환: ① 탐사 회차(전체/1차/2차/…) ② 센서(pH/EC).
  서버 불필요, 파일 하나로 공유 가능.
- PNG는 전체 누적 × 센서별 1장씩 (보고서용 기본).

### 대시보드

- 서버 시작 시 `data/measurements.csv` 를 읽어 회차별 집계를 메모리에 보관
  (표준 라이브러리 `csv` 모듈 — 서버는 pip 의존성 없음 유지).
- `/api/heatmap?sensor=<id>&run=<live|all|1|2|…>` 로 확장.
  `live`(기본) = 실시간 누적, `all`/숫자 = CSV 전체 또는 해당 회차.
- `/api/runs` : 회차 목록(번호, 날짜, 측정 수) 반환 — 드롭다운 구성용.
- 히트맵 카드에 "탐사 회차" 드롭다운 추가 (실시간 / 전체 누적 / 1차 / 2차 / …).
- CSV 파일이 없으면 회차 드롭다운은 "실시간"만 표시 (오류 아님).

## 기능 4 — Excel 수치 요약 내보내기

- 새 스크립트 `export_report.py` (openpyxl 사용):
  - 입력: `data/measurements.csv`
  - 출력: `output/report.xlsx` — 시트 3개
    1. **회차별 요약**: run_id, 측정일, 측정 수, pH 평균/최소/최대,
       EC 평균/최소/최대, 오염 셀 수, 최고 오염 위치(X, Y, 수심)
    2. **오염 셀 목록**: run_id, x_m, y_m, depth_m, 평균 pH, 평균 EC,
       측정 횟수, 판정 사유(pH/EC/둘 다) — 심한 순 정렬
    3. **셀별 전체 데이터**: 회차×셀별 평균값 전체 (수치화된 원자료 정리본)
  - 오염 셀 행은 빨간 배경으로 강조.
- 대시보드 히트맵 카드에 **"Excel 내보내기" 버튼** → `/api/export?run=<현재 선택>` 이
  현재 보고 있는 집계를 xlsx로 다운로드. 서버에 openpyxl이 없으면 CSV로 대체 응답.
- `.claude/skills/report/SKILL.md` 추가 — "보고서 만들어줘"로 실행.

## 파생 변경

- `generate_data.py`: 탁도 대신 pH·EC 필드 생성 (기존 플룸 모델 재사용 —
  dashboard_server.py 의 read 공식과 동일한 분포로 생성해 화면 간 일관성 유지).
  수심 3단계(0.5/1.0/1.5 m) × 수심당 3표본(20초 간격, 1분) 프로파일 적용.
- `dashboard_server.py` 시뮬레이터: 수심 하강을 `DEPTH_LEVELS = [0.5, 1.0, 1.5]` +
  수심당 체류(`DEPTH_DWELL`)로 변경, `MAX_DEPTH = 1.5`. 플룸 수심 중심 이동.
- `dashboard.html` 수심 카드 문구를 "수심당 1분, 지점당 3분 측정"으로 갱신.
- `requirements.txt`: openpyxl 추가.
- README·스킬 문서의 컬럼 형식 안내 갱신.

## 검토한 대안

- **상대 기준(상위 10%)**: 기준값 관리는 편하지만 사용자가 절대 임계값 방식을 선택 → 기각.
- **회차별 HTML 파일 분리**: 파일 수 증가 → 드롭다운 내장안 채택.
- **CSV 내보내기**: 사용자가 Excel 선택 → xlsx (다중 시트·색 강조 가능이 장점).

## 테스트 / 검증

- `generate_data.py` → CSV 컬럼·값 범위(오염원 부근 pH 하강, EC 상승) 확인.
- `export_report.py` → xlsx 3개 시트, 오염 행 빨간 강조, 회차 수 확인.
- `visualize_heatmap.py` → PNG 육안 확인(빨간 오염 마커·범례·한글), HTML 드롭다운 전환 확인.
- 대시보드 → Playwright로 회차 전환, 막대 경사 강조, 빨간 오염 표시, Excel 다운로드 확인.
